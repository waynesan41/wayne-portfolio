Personal Project: Full-stack Web App + Chrome Extension

Project: handybookmark.com
Technologies Used: React, TypeScript, Tailwind, Node.js, Express.js, MongoDB, Chrome Extension API, Docker, Ubuntu Server

Overview

  The Handy Bookmark is a full-stack web application that enables users to organize their bookmarks by attaching titles, favicons, descriptions, and colorful tags. It also includes a Chrome extension, allowing users to seamlessly save bookmarks from any website directly to their account.

Features

  1. User Authentication:
    • Secure login and registration system using JWT (JSON Web Tokens) for authentication.
    • Email Verification: Send a verification email to confirm the user's email address.
    • Password Reset: Allow users to reset their passwords through a secure email link, enabling them to regain access to their accounts.
  2. Bookmarking:
    • Users can create, edit, and delete bookmarks and tags. All using restful APIs.
    • Option to categorize bookmarks with custom color tags (e.g., Docker, Reading, Food Recipes) and add descriptions.
  3. Bookmark Database:
    • A searchable database of bookmarks with tags, titles, and descriptions.
    • Users can submit their tags to attach with bookmarks.
  4. Chrome Extension:
    • Users can save bookmarks from any webpage by clicking the extension icon or using the right-click menu.
    • The extension captures the current web page information such as title, favorite icon, and URL, and then sends this information to the popup window ready for the user to add tags and a description.

Technical Implementation

  • Front-end:
    • Built using TypeScript and ReactJS to create a dynamic user interface.
    • Utilized React Router for navigation between components.
    • Styled with Tailwind CSS to create dark and light theme options with responsiveness to screen sizes.
  • Back-end:
    • Developed a RESTful API using Node.js and Express.js to handle requests for user authentication, bookmarks, and tags management.
    • MongoDB was used to store user data, token sessions, bookmarks data, and tags data, with proper indexing in each collection for optimal search queries.
  • Hosting:
    • Self-hosting on an Ubuntu server running on a $200 home laptop with 8GB of RAM, secured through a Cloudflare Zero Trust tunnel.
    • Using PM2 to manage Node.js
    • I utilize the Docker compose .yaml file to configure MongoDB for the replica set with multiple instances.
  • Chrome Extension:
    • Created using JavaScript and the Chrome Extension API, allowing users to interact with the web app seamlessly.
    • Implemented background scripts to manage local data storage and communication with the web app.

Challenges and Solutions

  • Hosting
    • I initially used an AWS EC2 instance for a simpler hosting setup but later opted for self-hosting as it is more cost-effective.
  • MongoDB
    • I use MongoDB Transactions to preserve data consistency and integrity when manipulating multiple collections in a single execution. Since transactions require multiple replica sets, I utilize Docker to create multiple MongoDB instances on a single physical server.
