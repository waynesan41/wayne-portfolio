Personal Project: Mobile App (Android)

Project Title: Food Inventory Tracker (Play Store)
Technologies Used: Flutter, Dart

Overview

  Food Inventory Tracker is a mobile app that helps users manage their groceries by allowing them to track, sort and search individual food items. This comprehensive tracking system allow users to manage their groceries efficiently to avoid waste.

Features

  1. Tracking Items
    • Users can create, edit, and delete food items.
    • Users can hide specific items from the main list, allowing them to declutter their inventory view without permanently removing items.
    • If an item is accidentally deleted, users can easily undo the action and restore it to the inventory.
  2. Image Input
    • Users can attach images to food items by capturing a photo directly from the camera or selecting one from the gallery for easy visual identification.
  3. Filter Items:
    • Quickly find specific items in the inventory using the search function.
    • Easily sort the inventory by date added or expiration date to prioritize items that need to be used first.
  4. UI Theme:
    • Users can toggle between dark and light themes, providing a customizable and comfortable viewing experience based on personal preference.

Technical Implementation

  • State Management:
    • Used Flutter’s Provider package for efficient state management, ensuring that the app remains responsive and updates the UI automatically based on changes to the inventory.
  • Local Database Storage:
    • Integrated SQLite using the sqflite package for local data storage, allowing users to store food items, descriptions, expiration dates, and image URIs.
  • Image Handling:
    • Implemented image capture via the device's camera and image selection from the gallery using the image_picker package.
  • Persistent Theme Switching:
    • Light and dark theme persistence using SharedPreferences, so the user's theme preference is remembered even after closing the app.

Challenges

  • State Management
    • One of the significant challenges I faced during the development of the app was deciding which state management library to use. Flutter offers various options, including Provider, flutter_bloc, Riverpod, and others. Each library has its own strengths and weaknesses, which took additional time to research and compare those libraries.

Conclusion

  The Food Inventory Tracker demonstrates my ability to build a cross-platform mobile application with state management, and local data storage solutions. Through features like camera and gallery access and theme persistence, I’ve created a app that can helps manage groceries items. This project showcases my expertise in Flutter development and emphasizes my ability to integrate various technologies to create a mobile app. The challenge of selecting the right state management solution ultimately enhanced my understanding of Flutter.
